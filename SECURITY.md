# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | Yes       |

## Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.**

To report a security vulnerability, email: **nikzadpars@gmail.com**

Include:
- Type of vulnerability (e.g., SQL injection, authentication bypass)
- Full path of the source file(s) related to the vulnerability
- Steps to reproduce
- Proof-of-concept code (if possible)
- Impact assessment

You will receive an acknowledgement within 48 hours and a detailed response within 5 business days.

## Disclosure Policy

- Vulnerabilities are addressed as quickly as possible
- Credit is given to security researchers who report responsibly
- CVEs are filed for confirmed high/critical vulnerabilities
